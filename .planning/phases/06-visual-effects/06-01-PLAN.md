**globals.css** -- Add glass tier CSS custom properties AFTER the existing theme variables but BEFORE the `@theme inline` block:

In `:root` (light mode), add:
```
--glass-t1-bg: rgba(255, 255, 255, 0.6);
--glass-t1-border-highlight: rgba(0, 0, 0, 0.08);
--glass-t2-bg: rgba(255, 255, 255, 0.5);
```

In `.dark`, add:
```
--glass-t1-bg: rgba(22, 28, 44, 0.4);
--glass-t1-border-highlight: rgba(255, 255, 255, 0.15);
--glass-t2-bg: rgba(22, 28, 44, 0.5);
```

Per locked decisions: Tier 1 = blur 20px, saturate 1.5, bg opacity 0.4 dark / 0.6 light. Tier 2 = blur 12px, saturate 1.2, bg opacity 0.5. Gradient border simulated via inset box-shadow for Tier 1 only (simpler than pseudo-element, universal support).

- When NOT scrolled: keep `bg-ink-deep` (solid, no glass). Remove the border-b border-border from the non-scrolled state to make the nav invisible at top, add it back in scrolled state.
- The transition-all duration-300 already handles the background transition smoothly.
- Per locked decision: solid bg-ink-deep at top, transitions to full glass after scrollY > 10.

**Hero.tsx** -- Add a new glass overlay panel wrapping the existing content div:
- Keep the existing `<section>` wrapper and `<Waveform />` unchanged.
- Replace the content `<div className="relative z-10 mx-auto max-w-6xl text-center">` with a glass overlay div:
  ```
  <div className="relative z-10 mx-auto max-w-3xl rounded-2xl bg-[var(--glass-t1-bg)] px-8 py-12 text-center backdrop-blur-[20px] backdrop-saturate-[1.5] shadow-[inset_1px_1px_0_0_var(--glass-t1-border-highlight)]">
  ```
- Note: narrowed from max-w-6xl to max-w-3xl for a contained glass panel feel. The glass panel wraps ALL content including h1, subtitle, demo area, and Coming Soon badge.

Do NOT animate backdrop-filter values. Set them statically and only transition background-color/opacity.
