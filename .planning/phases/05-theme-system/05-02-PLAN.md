    1. Remove the hardcoded constants:
       - Delete `const BRAND_BLUE = "#3D7EFF";`
       - Delete `const EDGE_COLOR_RGB = "255,255,255";`

    2. Add a `ThemeColors` type:
       ```tsx
       type ThemeColors = {
         edgeRgb: string;
         gradientStart: string;
         gradientEnd: string;
       };
       ```

    3. Add a `colorsRef = useRef<ThemeColors>` initialized with fallback dark values.

    4. Create a `resolveThemeColors` function that reads CSS custom properties:
       ```tsx
       const resolveThemeColors = useCallback((): ThemeColors => {
         const style = getComputedStyle(document.documentElement);
         return {
           edgeRgb: style.getPropertyValue('--theme-waveform-edge-rgb').trim() || '255, 255, 255',
           gradientStart: style.getPropertyValue('--theme-waveform-gradient-start').trim() || '#6BA3FF',
           gradientEnd: style.getPropertyValue('--theme-waveform-gradient-end').trim() || '#2563EB',
         };
       }, []);
       ```

    5. Add a MutationObserver useEffect that watches `document.documentElement` for class changes (the `.dark` class toggle from next-themes):
       ```tsx
       useEffect(() => {
         const observer = new MutationObserver(() => {
           const oldColors = colorsRef.current;
           const newColors = resolveThemeColors();
           lerpColors(oldColors, newColors, 300);
         });
         observer.observe(document.documentElement, {
           attributes: true,
           attributeFilter: ['class'],
         });
         colorsRef.current = resolveThemeColors();
         return () => observer.disconnect();
       }, [resolveThemeColors]);
       ```

    6. Implement `lerpColors` function for smooth 300ms color transition:
       - Parse RGB values from both old and new color strings
       - Use requestAnimationFrame loop with smoothstep easing over 300ms
       - Interpolate between old and new values, updating colorsRef.current each frame
       - Handle both hex (#6BA3FF) and RGB tuple (128, 128, 128) formats

    7. Update `resolveBarColor` to use gradient for center bars (per CONTEXT.md: gradient #6BA3FF -> #2563EB in BOTH modes):
       - Accept `ctx: CanvasRenderingContext2D`, `y: number`, `barHeight: number` as additional parameters
       - Inner 40%: create a `ctx.createLinearGradient(0, y, 0, y + barHeight)` with `colorsRef.current.gradientStart` at 0 and `colorsRef.current.gradientEnd` at 1
       - Return type changes from `string` to `string | CanvasGradient`
       - Outer 60%: use `colorsRef.current.edgeRgb` instead of hardcoded `EDGE_COLOR_RGB`

    8. Update `drawBars` to pass `ctx`, `y`, `barHeight` to `resolveBarColor`.

    9. Ensure the reduced-motion static frame also uses theme colors (resolveThemeColors on mount).
