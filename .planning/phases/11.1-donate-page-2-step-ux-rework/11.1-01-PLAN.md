<objective>
Rework `/donate` into a 2-step flow: Step 1 is a method-picker (two large hero cards: Credit card / Bitcoin), Step 2 is a single centered amount card whose "Support" CTA is the ONLY redirect trigger. Swap the off-brand navy CTA gradient for the brand Accent Blue gradient. Decouple amount selection from payment redirect — chips and custom input now only update local state; the CTA confirms.

**Purpose:** Closes UX gaps identified by the user after shipping Phase 11-05:
1. Two near-identical cards side-by-side felt redundant
2. Navy CTA gradient clashed with the rest of the site's Accent Blue identity
3. Chip-click-to-redirect made the "Support" CTA feel vestigial and the flow unclear

**Output:** A single-component client-side 2-step flow with correct brand colors, disabled-until-ready CTA semantics, and preserved Stripe/BTCPay integration unchanged.
</objective>

    ```typescript
    function sanitizeDigits(raw: string): string {
      return raw.replace(/[^0-9]/g, "").slice(0, 4);
    }

    function clampStep(current: string, delta: number): string {
      const parsed = parseInt(current || "0", 10);
      const base = Number.isFinite(parsed) ? parsed : 0;
      const next = base + delta;
      if (next < MIN_AMOUNT) return String(MIN_AMOUNT);
      if (next > MAX_AMOUNT) return String(MAX_AMOUNT);
      return String(next);
    }
    ```
