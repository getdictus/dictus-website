# Phase 11.1 Plan 01: 2-step donate flow rewrite + Accent Blue CTA + i18n keys Summary

**2-step /donate flow: method-picker then single amount card whose Support CTA is the sole redirect trigger, with brand Accent Blue gradient on CTA/tiles/card border and theme-aware text tokens for light+dark readability.**

## Accomplishments

- Replaced side-by-side twin cards with a Step 1 method-picker (Credit card + Bitcoin) + Step 2 single amount card parameterized by chosen method
- Decoupled amount selection from payment redirect: chips and custom input only update local state; a disabled-until-ready CTA is the sole redirect trigger
- Swapped the off-brand navy gradient for the brand Accent Blue gradient on the CTA, method tiles, and amount-card border
- Added 7 new i18n keys per locale under `Donate` (method_fiat_title, method_fiat_subtitle, method_bitcoin_title, method_bitcoin_subtitle, method_bitcoin_lightning_tag, change_method, amount_prompt) while preserving legacy keys for backward compatibility
- Preserved Phase 11-05 "Why support?" inline-bullets section across both step states, unchanged
- Fixed light-theme readability regression by replacing static `text-mist` with theme-aware `text-text-primary` / `text-white-70`
- Replaced the literal `←` Unicode back-arrow (rendering above DM Sans baseline) with Iconify `solar:arrow-left-linear` for reliable inline-flex alignment

## Decisions Made

- **2-step flow over side-by-side cards**: The previous layout felt redundant and the Support CTA was vestigial. Splitting pick-method from pick-amount gives the CTA a clear role and reduces visual noise
- **CTA as sole redirect**: Chip click only marks the chip as selected; custom input only updates state. Keeps the interaction legible ("first pick amount, then confirm") and aligns fiat and bitcoin paths under a single UX pattern
- **Brand Accent Blue on tiles + CTA + card border**: Reinforces brand identity on the most monetization-critical page without oversaturation (border uses rgba(61,126,255,0.25) at rest, stronger glow on hover)
- **Theme-aware tokens over static palette**: `text-mist` is a fixed light blue that disappears on the light theme background; swapping to `text-text-primary` / `text-white-70` makes copy readable in both themes
- **Iconify back arrow**: DM Sans rendered the Unicode `←` above the text baseline; an SVG icon inside `inline-flex items-center` aligns reliably

### Auto-fixed Issues (during human-verification)

**1. [Rule 1 - Bug] Light-theme readability regression on Donate page**
- **Found during:** Human verification on localhost
- **Issue:** `text-mist` (#DBEAFE static light blue) was invisible against the light-theme page background on chip labels, Why-support bullets, and the thank-you footer
- **Fix:** Swapped to theme-aware Tailwind tokens: chip labels use `text-text-primary` (black on light / white on dark); Why-support bullets + thank-you footer use `text-white-70`. 4 occurrences across DonateCards.tsx and donate/page.tsx
- **Files modified:** src/components/Donate/DonateCards.tsx, src/app/[locale]/donate/page.tsx
- **Verification:** Re-tested in both themes during verification; copy readable in light and dark
- **Committed in:** f4c77b6

**2. [Rule 1 - Bug] Back-arrow vertical alignment broken with DM Sans**
- **Found during:** Human verification on localhost
- **Issue:** The literal `←` Unicode character had inconsistent metrics in DM Sans and rendered above the back-link text baseline inside `inline-flex items-center`
- **Fix:** Replaced the glyph with Iconify `solar:arrow-left-linear` at 16x16 so flex centering aligns it with the label
- **Files modified:** src/components/Donate/DonateCards.tsx
- **Verification:** Back link visually centered with its text label across viewports
- **Committed in:** f4c77b6

**3. [User request during verification] Brand-blue tiles + accent-tinted amount card**
- **Found during:** Human verification on localhost
- **Issue:** User flagged that method tiles used off-brand navy gradient (#0D2040 -> #071020) while the top-right nav Support button already uses brand Accent Blue, making tiles feel disconnected from the page's brand presence. The amount-card border (`border-border`, semi-transparent white) was also too faint at rest
- **Fix:** Swapped tile gradient to Accent Blue (#3D7EFF -> #2563EB) to match the nav button. Swapped amount-card border from `border-border` to `border-[rgba(61,126,255,0.25)]` at rest + `hover:border-accent` with stronger glow. Brand blue now present at tiles + card + CTA without over-saturation
- **Files modified:** src/components/Donate/DonateCards.tsx
