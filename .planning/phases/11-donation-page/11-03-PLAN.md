1. **Prerequisites**
   - SIRET number for PIVI Solutions
   - IBAN for payout bank account
   - Email address for Stripe account

2. **Create Stripe Account**
   - Go to https://dashboard.stripe.com/register
   - Complete business verification (company type: PIVI Solutions, SIRET)

   - Enable live mode once verified

     - Create Payment Link from that Price
     - Settings: no quantity adjustment, no promo codes, no shipping
     - Copy the Payment Link URL

5. **Create Custom Amount Payment Link** (1 link)
   - Add a Price: one-time, "Customer chooses what to pay"
   - Suggested amounts: 5, 10, 25, 50 EUR
   - Minimum: 1 EUR, Maximum: 999 EUR
   - Create Payment Link from that Price
   - Copy the Payment Link URL

   - Replace PLACEHOLDER URLs with real Payment Link URLs:

   - Confirm payment flow: click link -> Stripe checkout -> payment success

**Title:** BTCPay Server Setup Guide — Contribution Dictus

**Sections:**

1. **Overview**
   - BTCPay Server is self-hosted Bitcoin payment processor
   - Two deployment options: Umbrel (recommended, 0 EUR/month) or Hetzner VPS (fallback, 4 EUR/month)
   - Uses phoenixd for Lightning Network (auto liquidity management)

2. **Option A: Umbrel + Cloudflare Tunnel (Recommended — 0 EUR/month)**
   - Prerequisites: Umbrel node running, Cloudflare account with domain
   - Step 1: Install BTCPay Server from Umbrel App Store
   - Step 2: Configure Cloudflare Tunnel
     - Install cloudflared on Umbrel (or use Cloudflare Tunnel Umbrel app if available)
     - Create tunnel: `cloudflared tunnel create btcpay`
     - Route tunnel to BTCPay local address (typically localhost:23001)
     - Create DNS record: `btcpay.getdictus.com` -> tunnel
   - Step 3: Verify public access at https://btcpay.getdictus.com

3. **Option B: Hetzner CAX11 VPS (Fallback — 4 EUR/month)**
   - Rent Hetzner CAX11 ARM VPS (4 EUR/month, 4GB RAM, 40GB disk)
   - SSH into server
   - Deploy BTCPay Server via Docker:
     ```bash
     export BTCPAY_HOST="btcpay.getdictus.com"
     export NBITCOIN_NETWORK="mainnet"
     export BTCPAYGEN_CRYPTO1="btc"
     export BTCPAYGEN_LIGHTNING="clightning"
     export BTCPAYGEN_ADDITIONAL_FRAGMENTS="opt-save-storage"
     . ./btcpay-setup.sh -i
     ```
   - Configure DNS: `btcpay.getdictus.com` -> server IP
   - Use `opt-save-storage` fragment for pruned Bitcoin node (saves disk space)

4. **Store Configuration (both options)**
   - Create admin account on first visit
   - Create a new Store: "Dictus Contributions"
   - Store Settings > General: Enable "Allow anyone to create invoice" (CRITICAL — form POST will fail without this)
   - Store Settings > General: Set default currency to EUR
   - Set up Bitcoin wallet:
     - On-chain: Create new wallet or import existing xpub
     - Lightning: Install phoenixd plugin (auto liquidity, officially supported by BTCPay)

5. **phoenixd Lightning Setup**
   - Install phoenixd from BTCPay Server plugins
   - phoenixd handles channel management and liquidity automatically
   - No manual channel opening needed
   - Verify Lightning is active in Store > Lightning > Connection status

6. **Update Config File**
   - Open `src/config/donate.ts`
   - Replace placeholder values:
     ```typescript
     export const btcpayConfig = {
       serverUrl: "https://btcpay.getdictus.com",
       storeId: "YOUR_ACTUAL_STORE_ID",
     } as const;
     ```
   - Find Store ID: Store Settings > General > Store ID field

7. **Testing**
   - Open `https://btcpay.getdictus.com/api/v1/invoices` in browser — should show API info
   - Test form POST: create a test invoice by submitting form with storeId + price + currency
   - Verify invoice page shows QR code for on-chain and Lightning
   - Test a small payment on mainnet or use testnet first

8. **Notes**
   - No API key needed for invoice creation (uses "Allow anyone to create invoice")
   - Store ID is public (not a secret) — safe to commit to repo
   - For webhooks (Phase 12), you will need BTCPAY_WEBHOOK_SECRET env var
   - phoenixd manages Lightning liquidity automatically — no manual intervention needed
