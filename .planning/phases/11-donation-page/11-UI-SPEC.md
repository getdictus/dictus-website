## Interaction Contract

### Amount Chip Selection (Fiat)

1. User clicks a chip (e.g. "10 EUR")
2. Chip visually highlights with `bg-accent border-accent text-white` for 150ms feedback
3. Browser navigates to Stripe Payment Link with amount pre-filled (same tab or new tab per Stripe link config)
4. No intermediate confirmation step

### Amount Chip Selection (Bitcoin)

1. User clicks a chip (e.g. "10 EUR")
2. Chip visually highlights briefly
3. New tab opens with BTCPay Server invoice for the EUR amount (BTCPay converts to BTC)

### Custom Amount (both cards)

1. User types amount in input field (number input, min=1, max=999)
2. User clicks "Contribuer" / "Contribute" button
3. Same redirect behavior as chips (Stripe link or BTCPay invoice)
4. Input validation: reject non-numeric, below 1, above 999. Show inline error text below input.

### Nav Pill Button

1. Visible on all pages at all viewport widths
2. On mobile: consider hiding text and showing only on larger screens, or keeping compact
3. Click navigates to `/[locale]/donate`

### Card Hover

1. Border transitions from `border` to `border-hi` color
2. Subtle glow shadow appears: `0 0 20px var(--color-glow-soft)`
3. Transition duration: 300ms

## Accessibility

- All interactive elements (chips, buttons, inputs) have min 44px touch target
- Chips use `role="link"` with `aria-label` describing the action ("Contribuer 10 EUR via carte bancaire")
- Custom amount input has associated `<label>` (visually hidden if needed)
- Card sections use `<section>` with `aria-labelledby` pointing to card title
- External links (Stripe, BTCPay) include `rel="noopener noreferrer"` and `target="_blank"`
- Focus states: `focus-visible:ring-2 focus-visible:ring-accent focus-visible:ring-offset-2 focus-visible:ring-offset-ink-deep`
- Color contrast: white text on accent (#3D7EFF) meets WCAG AA for large text (chip labels)
