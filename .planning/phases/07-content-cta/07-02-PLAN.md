**Step 3: Create AdaptiveCTA.tsx**

Create `src/components/Hero/AdaptiveCTA.tsx` as a `"use client"` component:

- Import `useState`, `useEffect` from React
- Import `useTranslations` from `next-intl`
- Import `QRCodeSVG` from `qrcode.react`
- Define `DeviceType = "iphone" | "desktop" | "other-mobile" | "unknown"`
- `detectDevice()` function: check `navigator.userAgent` for `/iPhone/` -> "iphone", `/Android|webOS|iPad|iPod|BlackBerry|Opera Mini|IEMobile/i` -> "other-mobile", else "desktop"
- Read `process.env.NEXT_PUBLIC_TESTFLIGHT_URL` into a const (build-time inlined)
- State: `const [device, setDevice] = useState<DeviceType>("unknown")`
- useEffect: call `setDevice(detectDevice())` on mount (empty deps array)
- Rendering logic:
  1. If no testflightUrl OR device === "unknown" (SSR/pre-hydration): render the EXACT current "Coming Soon" badge markup (copy from Hero.tsx lines 68-75), using `t("badge")`
  2. If device === "iphone": render `<a href={testflightUrl}>` styled as accent button: `inline-flex items-center gap-2 rounded-full bg-accent px-8 py-3 font-normal text-white transition-colors hover:bg-accent-hi`, text: `t("cta_testflight")`
  3. If device === "desktop": render a flex column with:
     - `<p>` with `t("cta_available_iphone")` in `text-sm font-light text-white-70`
     - QR code: `<div className="rounded-xl bg-white p-3"><QRCodeSVG value={testflightUrl} size={120} level="M" bgColor="#FFFFFF" fgColor="#0A1628" /></div>`
     - Text link: `<a href={testflightUrl}>` with `t("cta_testflight_link")` styled `text-sm text-accent underline underline-offset-4 hover:text-accent-hi`
  4. If device === "other-mobile" (Android, etc.): render `<p>` with `t("cta_available_iphone")` + text link to TestFlight URL (NO QR code -- can't scan own screen)
- The component must maintain the `mt-6` spacing from its parent context (add to wrapper or document for Hero integration)
