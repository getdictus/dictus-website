# Roadmap: dictus Landing Page

## Overview

Marketing landing page for dictus -- an offline voice dictation keyboard by PIVI Solutions. Premium design with light/dark theme, Liquid Glass glassmorphism, bilingual FR/EN. This milestone adds donation infrastructure (Stripe + BTCPay), webhook notifications (Telegram), Desktop platform preparation, and social links.

- v1.1 Polish & Differentiation — Phases 5-7 (shipped 2026-03-17)

## v1.3 Donate & Desktop

**Milestone Goal:** Add donation page (Stripe fiat + BTCPay Bitcoin), webhook notifications via Telegram bot, Desktop platform badges (Coming Soon), and X/Twitter social link.

### Phase 10: Social & Footer Quick Win
**Goal**: X/Twitter icon visible in the footer on every page, linking to https://x.com/getdictus
**Depends on**: Nothing
**Requirements**: SOC-01, SOC-02
**Success Criteria**:
  1. Footer displays 3 social icons in order: GitHub, X/Twitter, Telegram
  2. X/Twitter icon links to https://x.com/getdictus (opens in new tab)
  3. Icon uses the official X logo SVG, same size and style as existing icons
  4. i18n label `twitter_label` exists in FR and EN translation files
**Estimated plans**: 1

### Phase 11: Donation Page & Payment Setup
**Goal**: Visitors can access `/donate` and choose to contribute via Stripe (fiat) or BTCPay Server (Bitcoin)
**Depends on**: Nothing (can run in parallel with Phase 10)
**Requirements**: DON-01, DON-02, DON-03, DON-04, DON-05, PAY-01, PAY-02
**Success Criteria**:
  1. Visiting `/fr/donate` and `/en/donate` displays a bilingual donation page with two cards side by side (stacked on mobile)
  2. Fiat card shows 4 suggested amounts (5/10/25/50 EUR) as selectable chips + custom amount input, CTA redirects to Stripe Payment Link with pre-filled amount
  3. Bitcoin card has a CTA that opens a BTCPay Server invoice (Pay Button or direct link)
  4. Page uses glassmorphism cards with hover glow, consistent with existing design system
  5. Text says "Contribution" / "Contribute", not "Donation" / "Donate" in legal/fine print context
  6. Setup guides for Stripe account and BTCPay Server are documented in the issue or a SETUP.md
**Estimated plans**: 2-3

**Research findings (for planning):**
- Stripe: Payment Link with `allow_custom_amounts`, pre-fill via `?prefilled_amount=` (cents). Fees: 1.5% + 0.25 EUR (EU cards). Account needs SIRET + IBAN, 1-2 days verification.
- BTCPay: Pay Button is simplest (HTML snippet, zero backend). Pruned node works (10-15GB disk). phoenixd supported for Lightning. Cheapest infra: Cloudflare Tunnel from Umbrel (0 EUR) or Hetzner CAX11 (4 EUR/month).

### Phase 12: Webhook Notifications (Telegram)
**Goal**: Every donation (Stripe or BTCPay) triggers a formatted notification in a Telegram chat via a bot
**Depends on**: Phase 11 (payment infrastructure must exist)
**Requirements**: HOOK-01, HOOK-02, HOOK-03, HOOK-04
**Success Criteria**:
  1. POST to `/api/webhooks/stripe` with valid Stripe signature sends a Telegram message with method, amount, currency, date
  2. POST to `/api/webhooks/btcpay` with valid HMAC signature sends a Telegram message with method, BTC amount, fiat equivalent, date
  3. Invalid signatures are rejected with 400 status (no Telegram message sent)
  4. Environment variables documented: TELEGRAM_BOT_TOKEN, TELEGRAM_CHAT_ID, STRIPE_WEBHOOK_SECRET, BTCPAY_WEBHOOK_SECRET, BTCPAY_BASE_URL
**Estimated plans**: 1-2

### Phase 13: Desktop Platform Preparation
**Goal**: The site communicates that Dictus is available on Desktop (Mac, Windows, Linux) with Coming Soon badges, and existing mobile-only copy is updated
**Depends on**: Nothing (can run in parallel with Phase 11/12)
**Requirements**: DESK-01, DESK-02, DESK-03, DESK-04
**Success Criteria**:
  1. A download/platforms section displays tabs (macOS | Windows | Linux) with OS auto-detection via navigator.userAgent to pre-select the right tab
  2. Each tab shows a Coming Soon badge (disabled button with platform icon) — no active download links
  3. A `downloads.ts` config file centralizes all platform links, versions, and enabled/disabled state — ready to flip when builds exist
  4. Site copy no longer says "iOS only" or "mobile only" — updated to reflect multi-platform availability (iOS, Android, Mac, Windows, Linux)
  5. Existing sections (features, comparison table, etc.) are reviewed and any mobile-specific language is generalized
**Estimated plans**: 2

**Research findings (for planning):**
- Handy.computer uses tabbed interface with OS auto-detection (navigator.userAgent/platform), config-driven download links in a single file, GitHub Releases for binaries.
- For Coming Soon: replace active buttons with muted/disabled badges + platform icons (Apple, Windows, Tux).
